Goals of Requirements Analysis

Presentation Date: Dienstag, 16. Juli 2024

Introduction

Requirements analysis is a critical process in software development. This presentation outlines the key goals and strategies to achieve clarity and efficiency in this process.

Key Goals

  • Understanding: Define the function signature and test cases.
  • Clarity: Ensure the causal chain is well understood.
  • Implementation: Identify which function to implement clearly.

Achieving Clarity

  • Avoid response waiting times by defining the tuple early.
  • Consider prototyping if requirements are unclear.
  • Ensure clarity on the function to implement by the end of discussions.

Hierarchy Levels

Top-down analysis involves understanding the system from the outside to the inside, focusing on different levels from system to interaction.

Entry Points and Interactions

  • Understand the difference between trigger functions and entry points.
  • Focus on request and response data for each entry point.

Command Query Separation (CQS)

  • Command: Changes state and acts like a POST in web.
  • Query: Reads state without changing it, similar to a GET in web.
  • Advantages include separate testability and reusability.

UI-Driven Slicing

  • Define entry points through user stories and slicing hierarchy.
  • Challenges in identifying all possible interactions and entry points.

Conclusion

  • Clarity and structure are essential in requirements analysis.
  • Hierarchy and interaction identification are key components.
